# 生产检查清单

## 开工前

- 项目 ID 已固定。
- 主角、反派、重要配角都有 `visual_lock`。
- 主场景都有 `visual_lock`。
- 单集目标时长明确。
- 结尾有悬念。
- 画幅固定为 9:16 或其他明确比例。

## 生成前

- `manga-flow check` 通过。
- 模型配置已经填写，或确认当前只跑占位流程。
- `.env` 中 API Key 已配置。
- 输出目录可写。

## 生成后

- `shot_list.csv` 镜头数量合理。
- `storyboard.html` 能快速看懂整集节奏。
- 每个镜头都有图片提示词。
- 关键镜头有视频提示词。
- `subtitles.srt` 时间轴没有明显错位。
- `edit_plan.json` 中每个镜头都有资产路径。
- `qc_report.md` 没有阻塞性问题。

## 人工抽检

- 主角是否变脸。
- 发型、服装、标志物是否保持一致。
- 台词是否口语化。
- 前 3 秒是否有钩子。
- 结尾 5 秒是否促使用户追下一集。
- 是否存在版权、肖像权、声音授权风险。

## 后续自动质检方向

- 图片相似度检查角色一致性。
- 视频黑屏检测。
- 音频静音检测。
- 字幕错别字检测。
- 口型同步检测。
- 敏感内容检测。
- 成本和失败率统计。
