# AI 漫剧全自动生成工作流方案

## 目标判断

现阶段最稳妥的路线不是一开始追求“全 AI 视频长片”，而是先做“动态漫画式 AI 漫剧自动化”，再把少量关键镜头升级成图生视频。

原因很直接：

- 长剧情跨镜头角色一致性仍然是主要瓶颈。
- 动态漫画更容易量产、复用和返工。
- 图生视频适合用在打斗、变身、情绪爆发、转场和结尾钩子。
- 自动化系统最重要的是结构化，不是一次性生成一个好看的片段。

## 推荐三阶段

### 阶段 1：动态漫画 MVP

目标：跑通“项目设定 -> 剧本 -> 分镜 -> 提示词 -> 字幕 -> 剪辑清单 -> 质检报告”。

建议规格：

- 竖屏 9:16
- 单集 60-90 秒
- 12-20 个镜头
- 2-3 个角色
- 1-2 个主要场景
- 输出成片前，先输出结构化生产包

当前代码已经覆盖这一阶段的基础结构。

### 阶段 2：半自动生产

目标：选定模型后接入真实生成能力。

需要接入：

- LLM：剧本、分镜、提示词扩写
- 图片模型：角色卡、场景图、分镜图、封面
- 视频模型：关键镜头图生视频
- TTS：旁白和角色配音
- 音效/BGM：环境音、转场、情绪铺底
- 剪辑：自动拼接、字幕、片头片尾、封面导出

### 阶段 3：批量自动化

目标：输入选题或大纲，系统批量输出多集内容。

需要补齐：

- IP 设定库
- 角色一致性素材库
- 提示词版本管理
- 失败重试
- 成本统计
- 自动审核
- 人工复核队列
- 发布素材生成

## 标准生产流程

1. **IP 设定库**
   固定世界观、角色关系、服装、道具、禁用设定、画风和台词风格。

2. **单集策划**
   从题材模板生成本集钩子、冲突、反转和结尾悬念。

3. **分镜拆解**
   每个镜头必须包含：场景、角色、景别、动作、情绪、台词、时长、图片提示词、视频提示词、音频说明。

4. **视觉资产生成**
   先生成角色三视图、表情表、服装表、场景基准图，再生成单镜头分镜图。

5. **视频生成**
   普通镜头使用静态图加镜头运动；关键镜头使用图生视频。

6. **音频生成**
   每个角色固定声音设定，生成旁白、对白、音效和 BGM。

7. **自动剪辑**
   按 `edit_plan.json` 拼接画面、配音、字幕、BGM 和转场。

8. **自动质检**
   检查时长、缺失资产、字幕、黑屏、静音、角色一致性和违规风险。

## 当前代码输出物

运行 `manga-flow run` 后会生成：

- `script.md`：分镜脚本
- `storyboard.html`：可浏览的分镜看板
- `shot_list.csv`：可给制片/运营看的表格
- `shot_list.json`：机器可读镜头表
- `prompts/image/*.txt`：图片提示词
- `prompts/video/*.txt`：视频提示词
- `audio/voice_lines.json`：配音台词
- `audio/subtitles.srt`：字幕
- `edit/edit_plan.json`：剪辑时间线
- `assets/asset_manifest.json`：资产清单
- `reports/qc_report.md`：质检报告

## 关键原则

- 先锁角色，再扩剧情。
- 先结构化，再接模型。
- 普通镜头便宜稳定，关键镜头才用视频模型。
- 每一步都要有可重试的中间产物。
- 模型输出不要直接进入最终成片，必须经过质检和人工抽检。
